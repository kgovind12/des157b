Coding Components and Feasibility Report

Features of my project:

  1. Form to add activities: The website will open with a form that users can fill out to log their sustainable activities. It will need multiple options (or chips) that the user can select. For this, a multiselect JavaScript or CSS library will be useful. Some examples of libraries I can use are Choices.js and pureScriptSelect. I aim to complete this feature within the first 2-3 days of starting the project.

  2. Virtual plant: Users will be able to grow their own virtual plant whenever they add an activity. This growing animation should be visible, and can possibly be done through CSS. Since there are no in-built libraries for growing plants in JavaScript, creating a realistic growing plant might be a little challenging to accomplish. In case this doesn't seem feasible, I will have a meter that keeps rising as activities are added, and the goal will be when it reaches the top. Another alternative would be to draw a small plant and make it scale up using CSS as activities are added. This feature might take more time, and I plan to allow myself 4-5 days to achieve this.

  3. Community garden: This feature is to allow users to see other users' plants. This interface should display all the plants that were grown on that day. This can be made possible by storing each plant's height in a database, and pulling all the entries from the database when the community garden page is loaded. These entries can then be used to populate the page with plants, since we now know their heights. Using CSS and JavaScript, we can display the plants on the page and scale them to their stored heights. This also might be a more time-consuming feature, so I will allow myself 4-5 days for this as well.

    The community garden should also have a scrolling interface. There are a few JavaScript libraries that allow for this, such as Sweet-scroll.js. This can also be done through Vanilla JS by giving each plant an index and incrementing the indices as the scroll arrows are clicked.

  4. Popups: When each plant is hovered over, there should be a popup that displays with information about the activities that helped grow it. This popup can be styled in CSS, and the functionality of opening and closing a popup can be done in Vanilla JavaScript by changing its opacity on hover. There are also some libraries to achieve this functionality such as Bootstrap Popover or Popper.js. Given the time constraint for this project, it might be more feasible to go this route.

Additional feature I will work on if I have time:

  1. Achievements page: This will be a page with pure text for users to see their achievements. This might be a difficult task to accomplish, because I will need to calculate the collective achievements of users using some kind of formula. In terms of styling, it will require CSS to color the numbers or statistics in each line of text. For the scope of this project, this feature will remain a low priority since it might not fit within the time constraint.
